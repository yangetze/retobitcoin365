# 🧹 Deduplicate ServiceWorker registration

## 🎯 What
Removed the duplicate `registerSW()` function and its invocation from `precios.js`. Centralized the ServiceWorker registration logic by consistently including `script.js` with the `defer` attribute in the `<head>` of all HTML pages (`index.html`, `precios.html`, `dashboard.html`).

## 💡 Why
The exact same `registerSW` functionality existed in two separate places (`script.js` and `precios.js`). This duplication increased maintenance overhead. Removing it from `precios.js` and centralizing it in `script.js` prevents duplication and ensures that PWA features are initialized consistently across the entire application using a single source of truth.

## ✅ Verification
- Verified that all unit tests still pass successfully using `npm test`.
- Started the local server and verified that the `script.js` is correctly included with the `defer` attribute in the `<head>` of `index.html`, `precios.html`, and `dashboard.html`.
- Verified that the Service Worker is correctly registered across all pages without console errors.
- Verified that moving `script.js` to the `<head>` in `index.html` aligns it consistently with the rest of the application without breaking functionality.

## ✨ Result
Improved codebase maintainability by centralizing core initialization logic and reducing redundant code. The application's entry points are now more consistent and easier to reason about.
